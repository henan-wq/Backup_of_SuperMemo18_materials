Title: 03 引用来源

为什么需要来源？


在渐进阅读中，我们始终需要快速回忆起一个问题或一段文本的上下文。快速回忆起上下文的最简单方法是添加引用来源。当我们生成摘录和完形填空时，引用来源会被继承到子元素。从给定文本中产生的所有子元素都带有引用来源标记，我们将无需担心丢失问题的上下文。


例如：

问：他出生于[...](year)


没有上下文就无法回答。但是，以下问题已经很容易理解：

问：他出生于[...](year)

--------------------------

#标题：Barrack Obama

#出处：Wikipedia





为了加快学习速度，在渐进阅读过程中，上述问题应当顺理成章的替换为：

问：奥巴马出生于[...](year)


或者

问：奥巴马出生于[...](year)

--------------------------

#标题：Barrack Obama

#出处：Wikipedia




引用来源不保存在我们文章的HTML文件中，而是存储在引用来源注册表中（即，在单独的数据库中）。引用来源注册表也不包含引用来源中的文本。所有引用文本都保存在可用于全局文本搜索的文本注册表中。在早期版本的SuperMemo中，每个文本将保留其自己的引用副本。在新版的SuperMemos中，元素仅调用引用来源注册表中的相应引用来源，而引用来源注册表再调用文本注册表中的具体文本。通过这种形式，同一条引用来源可以被多个元素引用。这样可以大大节省我们的集合空间。更重要的是，我们在某个元素中更新引用来源之后，会自动同步到其他使用同一引用来源的元素中去。这样，我们无需在搜索和替换上浪费时间来纠正其他元素中引用不正确的情况。


示范


如果选择源文章的标题并按Alt +T（或者：HTML 部件菜单上的Reference >> Title ），则每个摘录都将标记上源文章的标题。如果使用Edit >> Web import >> All导入文章，则文章将自动生成一些基本的引用来源（如#标题，#链接，#日期等）。如果需要更多背景信息（例如，添加作者，期刊等），则可以使用导航栏上的引用连接按钮（）跳转到生成该摘录的源文章。在原始文章（相对摘录而言）中，该按钮将通过浏览器打开文章原始链接。



图：渐进阅读的截图。在了解温室效应的同时，学生摘录了一段片段，说：“ 与地球距离太阳相同距离的理想导热黑体的温度约为5.3°C。但是，由于地球反射约30％[ 5 ] [ 6 ]的入射阳光，这个理想化的行星的有效温度（发射相同辐射量的黑体的温度）约为-18°C。[ 7 ] [ 8 ]这个假设行星的表面温度为比地球的实际表面温度约14°C低33°C[ 9]。在实际表面温度和有效温度之间产生这种差异的机制是由于大气引起的，被称为“ 温室效应 ”。摘录的片段将继承右侧的插图以及文章引用来源。学生可以继续学习。通过按Enter键阅读另一篇文章，右边的图片存储在本地图像注册表中（在用户的硬盘上），可以重复使用以说明其他文章或问题。


“来源”系统亮点


  • 要将文本标记为引用来源字段，请使用HTML 部件菜单上的Reference子菜单（Reference >> Select或Alt+ Q）
  • 引用来源字段中的#Article，#Parent和#Concept group会自动添加，并且不会存储在参考注册表中。没有定义其他参考字段的元素中不会生成这些字段
  • 通过Alt +Q选项添加的引用来源会显示在引用来源字段中，可以从文本正文中删除（如果不再需要）
  • 将鼠标悬停在导航栏上的引用链接按钮（）上，可快速查看较长摘录中的参考。
  • 从用户的角度来看，与早期的SuperMemos相比，引用的处理方式略有不同。SuperMemo 2008或更高版本区分以下两种引用编辑类型：
    • 局部编辑仅影响当前元素并创建新引用来源记录的
    • 全局编辑，可在使用原始引用来源的所有元素中进行更改。
  • 请注意，所有通过摘录产生的元素都是原文章的子级元素。如果我们在调用某片段的原始上下文时遇到问题，可以随时通过按导航栏上的 Parent 按钮来对其进行回调。我们还可以使用引用链接按钮（）跳转到源文章，或者，如果已经到达源文章，还可以跳转到网络上的原始文章。
  • 如果我们添加了#Date引用来源，但是没设置相应的日期，则将使用当前日期和时间标记文本
  • AND-Search in SuperMemo works on texts, not on elements. This means that reference texts do not take part in AND-Search for the main body of text. This may result in false misses. In SuperMemo, texts and individual reference fields are all treated as separate texts and are all searched independently
  • 引用的格式可以通过样式表进行更改
  • 将HTML转换为纯文本不会影响引用的格式（即纯文本条目的引用可以由样式表单独设置格式）
  • 我们可以在引用来源区域中或在专用窗口中编辑引用来源，方法是从元素菜单中选择Reference >> Edit 。我们只需要在每个引用来源行的开头使用合法的引用来源字段标签（例如#Author：）。如果SuperMemo不确定我们所做的更改是仅应用于当前元素还是应用于引用的所有元素，它将询问我们
  • 我们可以快速修改（即设置，合并和删除）多个元素的引用来源。操作是：在元素浏览器，右键单击鼠标并选择：
    • Process browser> : Reference : Set reference以对元素的子元素设置相同的引用来源，
    • Process browser> : Reference : Merge reference以将引用来源字段添加到元素的子集，
    • Process browser> : Reference : Delete reference 以从所有子元素中删除引用来源字段
  • 引用使我们的HTML文件不再混乱。过去，引用的大小通常大于文本本身的大小
  • 引用注册表保留引用（请参见下文），并且它们的各个文本字段存储在文本注册表中
  • 引用是在加载时添加到HTML文本的，因此我们仍然可以像在SuperMemo的早期版本中那样，将引用保留在文本的底部。
  • 将现有引用添加到元素（例如，使用元素菜单中的Reference >> Link ）不会增加集合的大小

注意！ 请勿在表示引用来源区域的水平线下方添加自己的非引用来源的文本。这一区域是SuperMemo自动管理的区域。对该区域的任何修改将被视为对引用来源字段的更改。不符合该字段格式的更改将被丢弃而不会发出任何警告。




图： 引用来源可帮助我们快速恢复给定元素的上下文以及跟踪其来源并建立引文列表。图中，是有关睡眠和梦境的文章的摘录。蓝色标记了从显示的文本中摘录的内容。黄色标记用于Search >> Find elements（Ctrl + F）以查找包含该字符串的所有元素（包括该元素）。粉色标记表示引用来源区域（由＃Title ，＃Author，＃Date ，＃Source ，＃Article ，#Parent 和＃Concept group 字段组成），并传递给从该元素生成的所有子元素（摘录和完形填空）。



图： 引用保存在专用注册表中，而它们的各个文本字段（例如＃Title ，＃Author ，＃Date ，＃Source 等）存储在文本注册表中，因此可用用于全局文本搜索。在图中，参考注册表拥有71,791条记录。以黄色突出显示的是下载图像的引用。其余引用用来描述导入的文章。所选的《量子生物学》引用描述了2013年2月从《自然物理学》撰写的文章，并从该文章导入了内容。元素列表面板（右下）显示了从该文章生成的摘录卡片。所有这些摘录卡片共用相同的引用来源。


编辑引用来源


我们可以在SuperMemo的Commander中使用Reference>>Edit ，但是，我们也可以在引用来源区域（默认样式表中为粉红色）中编辑参考。我们可以删除引用字段，但是我们需要确定该更改是局部的（仅适用于该元素）还是全局的（对于使用此引用的所有元素）。我们将无法删除#Article，#Parent：或#Concept group字段，因为它们会自动添加到引用来源部分（即它们不是参考本身的一部分）。非法更改是所有不符合参考格式的更改，例如，不以参考字段标签开头的行，或以未知的参考字段标签开头的行（例如#Country）。如果不确定此过程如何发生作用，请将一篇文章从Wikipedia导入到新创建的集合中，创建一些摘录并进行编辑以查看引用的处理方式。


图片形式的引用来源


从网络或HTML文件导入时，会自动创建图像引用来源。自SuperMemo 17开始，URL保留在文本注册表中，由于文本可以自动重复使用，因此从一定程度上解决了冗余问题。从文章或Web中导入图片时，采用这种复杂的解决方案可能会使SuperMemo的速度变慢。但是，我们的集合不会因为重复使用相同的图片而使体积增大，自动防止重复导入，并将导入的图像替换为存储的原始图像（wiki缩略图替换为其高分辨率的原始图像）。


图像引用允许使用 Edit >> Find elements（Ctrl +F）在元素中搜索图片名称以及其他文本。


我们可以在引用来源注册表中找到图像引用 （从主菜单中选择 Search >> References）。或者，我们可以首先通过图像注册表找到图像（从主菜单中选择Search >> Images），然后从右键单击，从注册表菜单中选择 View >> Reference。


在引用来源注册表中，要查看与给定引用相关的图像，请单击窗口底部的“Go to ”。


要查看与给定图像/引用关联的所有元素，请单击注册表（图像或引用来源）底部的List。
